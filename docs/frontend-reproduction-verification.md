# 星图前端复刻级验证标准

本标准把 `my.bdfz.net` 的宇宙主视线与底部记录明细、`qx.bdfz.net` 的学科星群筛选、`hp.bdfz.net` 的年代星图和镜头手感作为同一所有者产品族的交互参照。目标不是复制截图，而是复刻其空间关系、信息密度和操作合同，并让课程演变数据成为视觉结构本身。

## 1. 主视线

- 首屏 100% viewport 是 Canvas 宇宙；不得出现营销 hero、长说明、卡片墙或页面 footer 抢占主视线。
- 桌面 1440×1000 与手机 390×844 截图中，星图必须覆盖除固定顶栏和底部双入口外的全部区域。
- 深空层至少包含静态星尘、银河雾带、星体光晕和低频流星；稳定模式关闭持续动画但保留关系结构。

## 2. 数据即构图

- 每一资料星来自 `/api/documents`，不得用装饰性假节点冒充资料。
- 横向位置是分段年代尺度，给 1950、1978、2001、2011、2017、2022 等改革阶段独立呼吸空间；纵深和色彩来自学科。
- 实心光核表示可引文资料；虚线空心环表示 OCR/正文未过门槛；考试评价使用金色语义。
- 谱系线只表示“同学科、同学段、年代相邻”，界面必须明确不将其宣称为正式替代关系。
- 跨学科线只表示同一改革年份/批次的元数据关系；概念星和概念关系只绘制已入库记录。

## 3. 星图内操作

- 拖动旋转、Shift/右键平移、滚轮/双指缩放、双击复位均需在真实浏览器生效，并带阻尼。
- 学科按钮是显隐开关。切换后可见节点计数必须变化；“全部学科”在同一星图浮层完成，不跳页。
- 年代按钮与滑杆直接裁剪星体，不跳时间轴页面。
- `/terms` 必须留在主星图并切换跨学科模式；`/subjects` 与 `/timeline` 旧链接只能兼容回到星图，不得恢复独立 tab。
- 点击星体显示右侧证据检查器；点击概念星显示概念关系检查器。

## 4. 合并工作台

- `/compare`、`/sources`、`/search` 共用“版本与资料”底部工作台，内部仅有“版本比较 / 资料检索”两个栏目。
- `/ai`、`/discussions` 共用“教师研究”底部工作台，内部仅有“AI 研究 / 教师讨论”两个栏目。
- 工作台从底部进入，背景星图保留且降焦；关闭后回到原星图状态。
- 文档详情也使用同一工作台，不创建白底独立页面。

## 5. 证据边界

- 没有编辑核验结论的学科只显示版本序列并明确“摘要待核”，不得由前端推断变化。
- 全文检索和 AI 只返回文档、段落双重 `citation_allowed=1` 的内容。
- OCR 警告、同版/异版关系和“不允许 AI 引用”在检查器及资料工作台可见。

## 6. 复刻验收

预览发布后必须完成：

1. 与 `qx`、`hp` 和本地 User Center 宇宙代码的桌面截图并排审查，确认主视线、密度、色轨和镜头合同一致。
2. 真实远端数据计数等于 `/api/meta`；隐藏一个学科后计数下降，恢复后回到原值。
3. 跨学科模式、年代筛选、关键词过滤、星体详情、四个合并路由逐项点击。
4. 1440×1000、390×844 均满足 `documentElement.scrollWidth === innerWidth`。
5. 生产域不得有本站 JavaScript 错误；preview 域因 User Center 未将 workers.dev 加入 CORS 而产生的共享登录探测错误必须单独注明，不能误判为星图错误。
6. `/api/health`、`/api/meta`、文档详情、检索、未登录 AI 401 与评论策略回归通过。

任何一项失败都不满足复刻级上线标准。
