# 星图前端复刻级验证标准

本标准把 `my.bdfz.net` 的宇宙主视线与底部记录明细、`qx.bdfz.net` 的学科星群筛选、`hp.bdfz.net` 的年代星图和镜头手感作为同一所有者产品族的交互参照。目标不是复制截图，而是复刻其空间关系、信息密度和操作合同，并让课程演变数据成为视觉结构本身。

## 1. 主视线

- 首屏 100% viewport 是 Canvas 宇宙；不得出现营销 hero、长说明、卡片墙或页面 footer 抢占主视线。
- 桌面 1440×1000、1280×720 与手机 390×844 截图中，星图必须覆盖整个 viewport；学科/检索/谱系与年代/研究入口分别收敛为左右两条窄轨，不得再出现横贯视线的底部导航栏。
- 深空层至少包含静态星尘、银河雾带、星体光晕和低频流星；稳定模式关闭持续动画但保留关系结构。

## 2. 数据即构图

- 每一星体来自经 `scripts/build-concept-evolution.mjs` 验证生成的 `subject × curriculum line × edition × year × concept` 观察节点；文档、页码与段落只作为节点背后的证据，不得再把文献记录直接画成星。
- 横向位置是分段年代尺度，给 1950、1978、2001、2011、2017、2022 等改革阶段独立呼吸空间；纵深和色彩来自学科。
- 实心光核表示至少有一条文档与段落双门槛证据；环形星表示图文已人工复核但禁止逐字引用；虚线星表示 OCR/来源文本仍待核。
- 谱系线只表示“同一概念、同一学科、同一课程线在当前已核语料中的下一次观察”，不得宣称正式替代、因果演进、首次出现或消失。
- 跨学科线只表示同一概念表面在同一年、不同学科文本中被同时观察到；语义性的改名、重构、拆分和合并必须由编辑双端核证后才可加入。

## 3. 星图内操作

- 拖动旋转、Shift/右键平移、滚轮/双指缩放、双击复位均需在真实浏览器生效，并带阻尼。
- 左侧工具轨的固定顺序是“学科按钮列 → 星图检索 → 历代谱系/跨学科/概念深挖”；不得另加冗余的“全部学科”按钮。学科按钮区可独立滚动，检索与谱系工具不得被长学科名挤出 viewport。
- 学科按钮是显隐开关。非 Shift 单击隔离一个学科后，镜头必须只按当前可见节点自动适配，自动缩放上限为 `1.32`；再次单击恢复全部学科后按全图重新适配，上限恢复为 `1`。Shift 多选只改显隐，不强迫重置用户镜头；`/subjects?subject=...` 兼容路由必须执行同样的单学科适配。
- 右侧工具轨的固定顺序是“年代按钮与纵向年份滑杆 → 版本・资料 → 研究・讨论”；年代控件直接裁剪星体，不跳时间轴页面，两个入口不得以全宽底栏遮住 Canvas 或截获底部拖动。
- 星图检索同时覆盖已发布本体节点的标签、定义、`source_terms`、节点类型和版本范围；命中概念、官方术语或能力描述时，结果必须在主星图内形成可点击星群，不得跳到独立 tab。检索字段只用于定位，不能绕过节点自身证据与版本边界。
- “概念深挖”不得改变当前学科筛选。当前仅语文首批达到段落级深层本体发布门槛；选择其他学科时必须保留该学科星图并显示 fail-closed 空态，直到同学科、同版本的概念—术语—能力证据链核验完成，绝不能静默切回语文或伪造下位节点。
- `/terms` 必须留在主星图并切换跨学科模式；`/subjects` 旧链接兼容回到星图，`/timeline` 旧链接只可改写为 `/archive` 资料工作台，不得恢复独立 tab 或第二条可视化时间轴。
- 点击概念星显示右侧观察与原文证据检查器；OCR 候选只显示物理页和警告，不产生可引文链接。
- 134 个历史篇目只进入 `/archive` 和证据检查器；`century-observation-layer.json#star_projection` 的 OCR 候选 episode 必须与正式概念 episode 合并到同一 Canvas，使用同一年代坐标和右侧年份筛选。
- OCR 批次完成后，新增合格候选仍应通过确定性构建持续进入同一星图；不得用文档卡、第二 Canvas 或横向 timeline 代替星体投影。

## 4. 合并工作台

- 右侧“版本・资料”入口打开 `/compare`、`/sources`、`/search`、`/archive` 共用的底部工作台，内部保留“版本比较 / 资料检索 / 百年资料”栏目。
- 右侧“研究・讨论”入口打开 `/ai`、`/discussions` 共用的底部工作台，内部仅有“AI 研究 / 教师讨论”两个栏目。
- 工作台从底部进入，背景星图保留且降焦；关闭后回到原星图状态。
- 文档详情也使用同一工作台，不创建白底独立页面。

## 5. 证据边界

- 没有编辑核验结论的学科只显示版本序列并明确“摘要待核”，不得由前端推断变化。
- 全文检索和 AI 只返回文档、段落双重 `citation_allowed=1` 的内容。
- OCR 警告、同版/异版关系和“不允许 AI 引用”在检查器及资料工作台可见。

## 6. 复刻验收

预览发布后必须完成：

1. 与 `qx`、`hp` 和本地 User Center 宇宙代码的桌面截图并排审查，确认主视线、密度、色轨和镜头合同一致。
2. 星体计数等于 `concept-evolution.json` 的筛选结果；非 Shift 单击隔离学科后节点与两端关系同步下降，镜头只包围可见节点且缩放不超过 `1.32`；恢复全部后回到全图边界，Shift 多选不得强迫镜头复位。
3. 用一个 `definition` 命中词和一个 `source_terms` 命中词分别检索，确认结果以星群留在 Canvas 主视线内，点击后进入同一证据检查器；无证据节点不得因检索而升级引文资格。
4. 跨学科模式、年代筛选、星体详情、`/subjects?subject=语文` 与四个合并工作台路由逐项点击；右侧纵向滑杆位于两个工作台入口之前。
5. 1440×1000、1280×720、390×844 均满足 `documentElement.scrollWidth === innerWidth`，左右轨互不重叠，页面不存在全宽底部 dock，Canvas 底部仍可拖动。
6. 生产域不得有本站 JavaScript 错误；preview 域因 User Center 未将 workers.dev 加入 CORS 而产生的共享登录探测错误必须单独注明，不能误判为星图错误。
7. `/api/health`、`/api/meta`、文档详情、检索、未登录 AI 401 与评论策略回归通过。

任何一项失败都不满足复刻级上线标准。
