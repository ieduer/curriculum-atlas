# 贡献指南

本项目接受资料补充、版本关系修正、OCR 复核、无障碍改进与代码修复。所有内容贡献都必须可追溯、可复核，不得把搜索摘要、模型记忆或无法确认版本的转载文字当作原文。

## 开发流程

1. Fork 仓库并从 `main` 建立分支。
2. 使用 `npm ci` 安装依赖。
3. 只提交可公开再分发的代码、元数据和短引文；不要提交 PDF、数据库导出、用户数据、日志或凭据。
4. 运行 `npm run verify`。
5. Pull request 说明来源、版本身份、变更范围、验证结果与回滚方式。

## 资料与 OCR 贡献

- 为新增文件提供正式名称、发布/实施年份、发布机关、阶段、学科、文件类型、来源 URL、获取日期及版本关系。
- 扫描资料必须保留源 SHA-256、页码、引擎与处理版本。
- OCR 校正必须遵循 `docs/ocr-quality.md`：扫描图像是第一证据；同文同版在线文本才可校正文句；异版只能旁证稳定事实。
- 无法消除的冲突必须标记不确定性，并保持 `citation_allowed=false`。

安全问题不要提交公开 issue，请按 `SECURITY.md` 私下报告。
